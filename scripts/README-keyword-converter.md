# 关键字转双链转换脚本使用指南

## 功能介绍

这个脚本可以帮助您自动将文本中的关键字转换为Obsidian双链格式，实现智能链接创建。

## 核心功能

### 1. 关键字识别
- 自动识别文本中的关键字（基于大小写敏感匹配）
- 支持匹配页面的 `title` 和 `aliases` 字段
- 优先使用 `title` 匹配，其次是 `aliases`

### 2. 匹配规则
- **大小写敏感**：`Spark` 匹配 `Spark`，但不匹配 `spark`
- **精确匹配**：`Apache Spark` 不会匹配 `Spark`
- **多别名支持**：一个页面可以有多个别名

### 3. 转换方式
- **单个转换**：选中文字或光标位置单词
- **批量转换**：整个文件中的所有匹配关键字

## 使用方法

### 前置条件

1. **安装Dataview插件**（推荐）
   - 脚本会优先从Dataview获取页面信息
   - Dataview提供了更高效的页面数据访问

2. **配置页面元数据**
   确保您的笔记包含正确的frontmatter：

```markdown
---
title: Apache Spark
aliases:
  - Spark
  - ApacheSpark
date: 2023-01-01
---

# Apache Spark

这是一个关于Spark的文档...
```

### 安装脚本

1. 将 `simple-keyword-converter.js` 文件放入您的Obsidian `scripts` 文件夹
2. 如果没有scripts文件夹，请手动创建

### 使用方式

#### 方法一：命令面板（推荐）

1. 打开一个笔记文件
2. 按 `Cmd+P` (Mac) 或 `Ctrl+P` (Windows/Linux) 打开命令面板
3. 搜索以下命令：
   - **转换选中文字为双链**：转换当前选中的文字
   - **批量转换文件关键字为双链**：转换整个文件中的匹配关键字

#### 方法二：快捷键设置

您可以自定义快捷键：
1. 打开设置 → 快捷键
2. 搜索 `keyword-converter`
3. 为两个命令设置快捷键

## 使用示例

### 场景1：单个单词转换

假设您有以下页面：
```markdown
---
title: Apache Spark
aliases:
  - Spark
---
```

在笔记中选中 `Spark` 并运行脚本，会自动转换为：
```markdown
[[Apache Spark]]
```

### 场景2：批量转换

假设您有以下内容：
```markdown
今天我们来学习Spark框架。Spark是一个强大的大数据处理引擎。

另外，Flink也是一个很好的选择。Hadoop则是传统的大数据处理方案。
```

运行批量转换后可能变成：
```markdown
今天我们来学习[[Apache Spark]]框架。[[Apache Spark]]是一个强大的大数据处理引擎。

另外，Flink也是一个很好的选择。Hadoop则是传统的大数据处理方案。
```

### 场景3：多别名匹配

假设您有以下页面：
```markdown
---
title: Apache Spark
aliases:
  - Spark
  - 大数据Spark
  - Spark大数据
---
```

当您选中不同的关键字时：
- `Spark` → `[[Apache Spark]]`
- `大数据Spark` → `[[Apache Spark]]`

## 匹配优先级

脚本的匹配优先级如下：

1. **title匹配**（最高优先级）
   - 直接匹配页面的title
2. **alias匹配**
   - 匹配页面定义的aliases
3. **精确匹配**
   - 关键字必须完全匹配

## 注意事项

### 大小写敏感
脚本是大小写敏感的，这意味着：
- ✅ `Spark` → `[[Apache Spark]]`（匹配成功）
- ❌ `spark` → `[[Apache Spark]]`（匹配失败）

### 性能考虑
- 批量转换时，大型文件可能需要一些时间
- 建议在最终版本前先在副本文件上测试

### 错误处理
- 如果找不到匹配，脚本会在控制台显示提示
- 如果Dataview不可用，脚本会尝试从文件系统读取数据

## 故障排除

### 问题1：找不到页面数据
**症状**：提示"未找到Dataview页面数据"

**解决方案**：
1. 确保已安装Dataview插件
2. 确保Dataview插件已启用
3. 检查页面是否包含frontmatter

### 问题2：转换失败
**症状**：提示"未找到匹配的关键字"

**解决方案**：
1. 确保关键字的大小写与title/aliases完全一致
2. 检查页面是否包含对应的title或aliases
3. 使用批量转换前，先尝试单个转换

### 问题3：批量转换效果不理想
**症状**：转换结果不符合预期

**解决方案**：
1. 检查是否有多个页面包含相同的关键字
2. 使用alias字段来区分不同的关键字
3. 考虑调整页面的title和aliases

## 高级技巧

### 自定义别名
可以通过aliases字段来支持多个匹配：
```yaml
---
title: Apache Spark
aliases:
  - Spark
  - Spark大数据
  - 大数据处理Spark
---
```

### 避免过度转换
有些单词可能不需要转换为双链，可以通过以下方式处理：
1. 在页面中定义更具体的别名
2. 使用不同的命名约定
3. 手动调整转换结果
